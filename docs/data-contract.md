# Контракт данных

## Принципы

1. Сырые источники не считаются проверенной памятью.
2. Сводка без ссылки на источник считается гипотезой.
3. Конфликт между каналами должен сохраняться как конфликт, а не сглаживаться.
4. Запись во внешние системы запрещена без явного режима write-back.
5. Секреты не являются контекстом и не хранятся в `context/`.

## Статусы проверки

| Статус | Значение |
| --- | --- |
| `raw` | импортировано без обработки |
| `parsed` | извлечено в структуру |
| `summarized` | сжато в сводку |
| `verified` | сверено с источником |
| `stale` | устарело или требует перепроверки |
| `conflict` | противоречит другому источнику |

## JSONL-схемы

### Решение

```json
{"id":"decision-001","date":"2026-05-07","source":"gmail/thread-id","project":"example","decision":"...","confidence":"verified","links":["..."]}
```

### Задача

```json
{"id":"task-001","source":"linear/ABC-123","project":"example","owner":"...","status":"open","title":"...","due":null,"links":["..."]}
```

### Проект

```json
{"id":"project-001","name":"example","sources":["github/repo","notion/page"],"status":"active","summary":"..."}
```

### Человек или организация

```json
{"id":"person-001","name":"...","role":"...","organization":"...","sources":["gmail/contact"],"notes":"..."}
```

## Правило ссылок

Каждый нормализованный факт должен хранить:

- `source`;
- дату или диапазон;
- статус проверки;
- ссылку или путь;
- краткое объяснение, почему факт важен.
