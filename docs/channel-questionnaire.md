# Опросник каналов для Mirage

Цель: определить, какие источники личного и рабочего контекста нужно подключить к Mirage, с какими правами, за какой период и для каких сценариев.

## 1. Базовый профиль

1. Какие 3-7 рабочих направлений сейчас важны?
2. Какие проекты должны быть видны агенту всегда?
3. Какие проекты, люди или темы запрещены для обработки?
4. На каком языке агент должен хранить сводки: русский, английский, смешанный режим?
5. Что считать источником истины при конфликте данных: документы, задачи, письма, чаты, репозитории?

## 2. Каналы коммуникации

### Gmail / Email

1. Используете ли Gmail, корпоративную почту, IMAP/SMTP или несколько ящиков?
2. Какие папки или labels важны: inbox, sent, starred, project labels, newsletters?
3. Нужны ли только чтение и поиск или также черновики/ответы?
4. Какой период индексировать первым: 7 дней, 30 дней, 90 дней, все?
5. Какие отправители или домены исключить?

### Slack

1. Какие workspaces используются?
2. Какие каналы важны: product, engineering, founders, incidents, sales, support?
3. Нужны ли private channels?
4. Нужны ли файлы и изображения из сообщений?
5. Нужен ли write-back: создание сообщений, тредов, задач?

### Telegram

1. Telegram используется для работы, личных заметок, комьюнити или поддержки?
2. Какие чаты и каналы важны?
3. Нужны ли вложения: voice, images, files?
4. Есть ли каналы, которые агенту нельзя читать?
5. Нужно ли собирать личные сообщения или только группы/каналы?

### Discord

1. Какие серверы и каналы важны?
2. Нужна ли история сообщений или только свежие события?
3. Есть ли вложения и thread discussions, которые важны?
4. Нужна ли связь с GitHub/Linear/Notion задачами?

## 3. Документы и знания

### Google Drive / Docs / Sheets / Slides

1. Какие папки Drive являются источниками истины?
2. Какие Docs являются evergreen-документами?
3. Какие Sheets содержат операционные данные?
4. Какие Slides нужны для позиционирования, продаж, инвесторов или продуктовых решений?
5. Нужен ли write-back: комментарии, правки, создание сводок?

### Notion

1. Какие workspaces и базы данных используются?
2. Какие страницы являются активными: roadmap, specs, CRM, meeting notes, wiki?
3. Какие базы нужно читать как таблицы?
4. Разрешены ли правки Notion агентом?
5. Какие страницы считать архивом?

### Локальные файлы

1. Какие локальные папки кроме этой нужно подключить через `disk`?
2. Есть ли Obsidian vault, Desktop folders, project docs?
3. Какие форматы важны: md, pdf, pptx, xlsx, csv, json, images, audio?
4. Нужна ли транскрибация audio/video?

3. Нужен ли write-back: создание задач, обновление статусов, комментарии?
4. Какие поля обязательны при создании задачи агентом?

2. Нужны ли продуктовые события из PostHog?

## 7. Политика доступа

1. Какие источники можно читать без подтверждения?
2. Какие источники требуют подтверждения перед каждым чтением?
3. Какие источники запрещены для записи?
4. Какие действия агент может делать сам: поиск, сводка, индексация, snapshot?
5. Какие действия требуют явного подтверждения: отправка сообщений, изменение задач, правка документов, чтение production DB?

## 8. Выходные артефакты

Выберите, что нужно получать после синхронизации:

- ежедневный индекс новых событий;
- список решений;
- список задач и обязательств;
- карта проектов;
- карта людей и организаций;
- сводка рисков;
- changelog по репозиториям;
- список документов без владельца;
- найденные противоречия между каналами;
- краткий briefing перед рабочим днем.

## Критическая проверка

Самое слабое место будущей системы не Mirage, а границы доступа. Если подключить все каналы сразу без правил, агент получит шум, риск утечек и конфликтующие источники истины. Начинать нужно с 2-3 read-only каналов и явной схемы: что читается, что индексируется, что считается устаревшим, что запрещено.
